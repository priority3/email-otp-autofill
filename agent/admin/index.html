📨 Outlook OAuth

在 Azure 注册一个支持「任何 Entra ID 租户 + 个人 Microsoft 账户」的应用，把它配置为 Mobile/Desktop public client， 再把 Application (client) ID 填入此处。所有用户的 Outlook 授权登录共用它。
📌 获取步骤：打开 Azure 门户 · 应用注册 → New registration → 支持的账户类型选「任何组织目录(任何 Entra ID 租户)中的账户和个人 Microsoft 账户」→ 注册后进入 Authentication → 添加平台(Add a platform) → 移动和桌面应用程序(Mobile and desktop applications) → 添加重定向 URL https://login.microsoftonline.com/common/oauth2/nativeclient → 将「Allow public client flows」设为 Yes → 复制「Application (client) ID」填入下方。
Google OAuth（Gmail）
在 Google Cloud Console 注册一个 OAuth 2.0 Web 应用，将 Client ID 和 Client Secret 填入此处。 所有用户的 Gmail OAuth 授权共用它。
📌 获取步骤：打开 Google Cloud Console · Credentials → Create Credentials → OAuth client ID → Application type 选 Web application → Authorized redirect URIs 添加 https://<your-agent-host>/v1/gmail/auth/callback（如使用扩展浏览器流程则留空，扩展会自动处理）→ 复制 Client ID 和 Client Secret 填入下方。
